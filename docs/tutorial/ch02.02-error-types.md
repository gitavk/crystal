## 2.2 Define error types in crystal-core

Before building the K8s client, we need typed errors so the app can distinguish
"no kubeconfig" (show a warning) from "cluster unreachable" (retry later).

Update the crystal-core crate dependencies first:

**crates/crystal-core/Cargo.toml:**

```toml
[package]
name = "crystal-core"
version = "0.1.0"
edition = "2021"

[dependencies]
anyhow.workspace = true
kube.workspace = true
k8s-openapi.workspace = true
futures.workspace = true
tokio.workspace = true
tokio-util.workspace = true
tracing.workspace = true
serde.workspace = true
jiff.workspace = true
```

**crates/crystal-core/src/error.rs:**

```rust
use std::fmt;

// Custom error type for Kubernetes operations.
// Having distinct variants lets the UI show appropriate messages:
//   NoKubeconfig     → "No cluster connection" (non-fatal)
//   ConnectionFailed → "Cluster unreachable" (retry with backoff)
//   ApiError         → "API error: ..." (show in resource pane)
//   WatchError       → "Watch error: ..." (watcher will auto-retry)
#[derive(Debug)]
pub enum KubeError {
    NoKubeconfig,
    ConnectionFailed(String),
    ApiError(String),
    WatchError(String),
}

// Display is required for printing errors to logs and UI.
impl fmt::Display for KubeError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            Self::NoKubeconfig => write!(f, "No kubeconfig found"),
            Self::ConnectionFailed(msg) => write!(f, "Connection failed: {msg}"),
            Self::ApiError(msg) => write!(f, "API error: {msg}"),
            Self::WatchError(msg) => write!(f, "Watch error: {msg}"),
        }
    }
}

// Implementing std::error::Error lets KubeError interop with anyhow and other
// error-handling crates. The default implementation is enough — it uses Display
// for the message and has no source error.
impl std::error::Error for KubeError {}
```

Now update lib.rs to declare all the modules we'll create in this chapter:

**crates/crystal-core/src/lib.rs:**

```rust
// Declare the five new modules. Rust will look for files with matching names
// in the same directory (client.rs, context.rs, error.rs, informer.rs, resource.rs).
pub mod client;
pub mod context;
pub mod error;
pub mod informer;
pub mod resource;

// Re-export the most commonly used types so consumers can write
// `crystal_core::KubeClient` instead of `crystal_core::client::KubeClient`.
pub use client::KubeClient;
pub use context::{ClusterContext, ContextResolver};
pub use error::KubeError;
pub use resource::{PodPhase, PodSummary, ResourceSummary};

// The Command enum from Chapter 1 — still here, will grow in later chapters.
pub enum Command {
    Quit,
}
```
