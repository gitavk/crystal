## 2.1 Add Kubernetes dependencies to the workspace

[<< Previous](ch02.00-kubernetes-core.md) | [Contents](ch02.00-kubernetes-core.md) | [Next >>](ch02.02-error-types.md)

---

Kubernetes support requires three main crates plus two helpers. All go into the
workspace root so every crate can share the same versions.

```toml
# Cargo.toml (workspace root) — add these lines to [workspace.dependencies]

# kube-rs: the Rust Kubernetes client. "client" enables API calls, "runtime"
# enables watchers and informers, "derive" enables custom resource macros.
kube = { version = "3.0", features = ["client", "runtime", "derive"] }

# k8s-openapi: auto-generated Rust types for every Kubernetes API object
# (Pod, Namespace, Deployment, etc.). "latest" tracks the newest K8s version.
k8s-openapi = { version = "0.27", features = ["latest"] }

# futures: utilities for working with async streams (used by the watcher).
futures = "0.3"

# jiff: a time library. k8s-openapi 0.27 uses jiff::Timestamp internally
# for all timestamp fields (creation_timestamp, etc.).
# Note: older k8s-openapi versions used chrono — 0.27 switched to jiff.
jiff = "0.2"

# tokio-util: provides CancellationToken for cleanly stopping background tasks.
tokio-util = { version = "0.7", features = ["rt"] }
```

**Version note:** The kube crate went through a major version bump. Version 3.0
(released January 2026) changed the watcher `Event` enum to use
`Apply`/`Delete`/`Init`/`InitApply`/`InitDone` instead of the older
`Added`/`Modified`/`Deleted`. If you see examples online using the old names,
they're from kube 1.x or earlier. Similarly, k8s-openapi 0.27 switched its
internal time representation from `chrono::DateTime` to `jiff::Timestamp`.
