### What you should see

With a running cluster:

- Status bar shows your cluster name and current namespace
- Pod list populates with real pods from the default namespace
- `j`/`k` moves the highlight, wrapping at the edges
- `:` opens the namespace selector popup
- Typing in the popup filters the namespace list
- `Enter` switches namespace and refreshes the pod list
- `Esc` closes the popup without changing anything
- `q` quits

Without a cluster (`KUBECONFIG` unset or invalid):

- App starts without crashing
- Body shows "Error: No cluster connection"
- Status bar shows "no cluster | ns:n/a"
- `q` still quits

### Data flow diagram

```
kubeconfig → KubeClient → ContextResolver (single source of truth)
                ↓
         inner_client()
                ↓
       ResourceWatcher (background tokio task)
                ↓
         mpsc::channel(16)
                ↓
    EventHandler.forward_kube_events()
                ↓
   AppEvent::KubeUpdate → App.handle_kube_update()
                ↓
     ResourceListState.set_items()
                ↓
    App.build_render_context() → RenderContext
                ↓
       layout::render_root() → Frame
```

Panes never talk to K8s directly — all data flows through App Core.

### Architecture notes

Key patterns established in this chapter:

1. **Summary types at the boundary**: Full K8s API objects (`Pod`) are converted
   to lightweight summaries (`PodSummary`) at the point of entry. The TUI never
   sees the full objects.

2. **Channel-based event merging**: The watcher runs in a background task and
   sends snapshots through a bounded channel. The `EventHandler` forwards these
   into the main event stream, so the main loop has a single `match` for all
   event types.

3. **RenderContext pattern**: The TUI crate accepts a borrowed `RenderContext`
   struct for rendering. This prevents circular dependencies — `crystal-tui`
   never imports from `crystal-app`.

4. **Deferred namespace switching**: Key handlers are synchronous but starting a
   watcher requires async context. The `pending_namespace_switch` field bridges
   this gap — the key handler sets the flag, the main loop processes it.

5. **Graceful degradation**: Every K8s error is caught and converted to a UI
   message. The app never crashes on cluster issues.
