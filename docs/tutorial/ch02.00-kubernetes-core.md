# Chapter 2 — Kubernetes Core Integration

This chapter connects Crystal to a real Kubernetes cluster using kube-rs, lists
namespaces and pods in a scrollable table, adds a namespace selector popup, and
establishes the data layer patterns that all future resource views will follow.

## Prerequisites

- Chapter 1 complete (TUI skeleton renders and quits)
- Access to a Kubernetes cluster (minikube, kind, or remote) — app runs without
  one too

## What you will build

A TUI that reads your kubeconfig, connects to the active cluster, and displays
pods in a color-coded table with real-time updates via a Kubernetes watcher.
You can navigate the list with `j`/`k`, open a filterable namespace selector
with `:`, and switch namespaces on the fly. If no cluster is available, the app
starts gracefully and shows a warning instead of crashing.
