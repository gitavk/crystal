## 2.3 Define resource summary types

[<< Previous](ch02.02-error-types.md) | [Contents](ch02.00-kubernetes-core.md) | [Next >>](ch02.04-context-resolver.md)

---

A core design principle: **never store full Kubernetes API objects in TUI
state**. Instead, convert them to lightweight summary structs at the boundary.
This keeps the TUI fast (small structs, cheap to clone) and decoupled from the
K8s API (if a field is added to the Pod spec, only the `From` impl changes).

**crates/crystal-core/src/resource.rs:**

```rust
use std::fmt;
use std::time::Duration;

// jiff::Timestamp is what k8s-openapi 0.27 uses inside its Time type.
// We use it to compute pod age: now() - creation_timestamp.
use jiff::Timestamp;
// The full Kubernetes Pod type — we only use it in the From impl, never in the TUI.
use k8s_openapi::api::core::v1::Pod;

// The phase of a pod's lifecycle. Kubernetes returns this as a string;
// we convert it to an enum for type safety and pattern matching.
#[derive(Debug, Clone, PartialEq)]
pub enum PodPhase {
    Running,
    Pending,
    Succeeded,
    Failed,
    Unknown,
}

// Display impl lets us convert PodPhase back to a string for the table.
impl fmt::Display for PodPhase {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            Self::Running => write!(f, "Running"),
            Self::Pending => write!(f, "Pending"),
            Self::Succeeded => write!(f, "Succeeded"),
            Self::Failed => write!(f, "Failed"),
            Self::Unknown => write!(f, "Unknown"),
        }
    }
}

// A lightweight summary of a Pod — just the fields needed for the table view.
// This is what the TUI stores and renders. Compare this to the full Pod type
// which has hundreds of fields across spec, status, and metadata.
#[derive(Debug, Clone)]
pub struct PodSummary {
    pub name: String,
    pub namespace: String,
    pub status: PodPhase,
    pub ready: String,        // "1/1", "2/3" — ready containers / total containers
    pub restarts: i32,
    pub age: Duration,
    pub node: Option<String>,
}
```

### The ResourceSummary trait

A trait that any resource type can implement for uniform table rendering.
When we add Deployments, Services, etc. in later chapters, they'll each
implement this trait with their own column definitions.

```rust
// Send + Sync bounds are required because these values cross async boundaries
// (sent through channels from the watcher task to the UI task).
pub trait ResourceSummary: Send + Sync {
    fn name(&self) -> &str;
    fn namespace(&self) -> Option<&str>;
    fn status_display(&self) -> String;
    fn age(&self) -> Duration;
    // Returns (header, value) pairs — the table widget reads these to build columns.
    fn columns(&self) -> Vec<(&str, String)>;
}

impl ResourceSummary for PodSummary {
    fn name(&self) -> &str {
        &self.name
    }

    fn namespace(&self) -> Option<&str> {
        Some(&self.namespace)
    }

    fn status_display(&self) -> String {
        self.status.to_string()
    }

    fn age(&self) -> Duration {
        self.age
    }

    fn columns(&self) -> Vec<(&str, String)> {
        vec![
            ("NAME", self.name.clone()),
            ("NAMESPACE", self.namespace.clone()),
            ("STATUS", self.status.to_string()),
            ("READY", self.ready.clone()),
            ("RESTARTS", self.restarts.to_string()),
            ("AGE", format_duration(self.age)),
            ("NODE", self.node.clone().unwrap_or_default()),
        ]
    }
}
```

### The conversion boundary: Pod → PodSummary

This is the critical boundary: full Kubernetes Pod → lightweight PodSummary.
The full Pod object is dropped after this conversion — it never reaches the
rendering layer.

```rust
// `From<&Pod>` means "I can be created from a reference to a Pod".
// We use a reference (&Pod) because we don't need to consume the original.
impl From<&Pod> for PodSummary {
    fn from(pod: &Pod) -> Self {
        let metadata = &pod.metadata;
        let name = metadata.name.clone().unwrap_or_default();
        let namespace = metadata.namespace.clone().unwrap_or_else(|| "default".into());

        // pod.status.phase is an Option<String> like "Running", "Pending", etc.
        // We map the string to our enum, defaulting to Unknown for unrecognized values.
        let status = pod
            .status
            .as_ref()
            .and_then(|s| s.phase.as_deref())
            .map(|p| match p {
                "Running" => PodPhase::Running,
                "Pending" => PodPhase::Pending,
                "Succeeded" => PodPhase::Succeeded,
                "Failed" => PodPhase::Failed,
                _ => PodPhase::Unknown,
            })
            .unwrap_or(PodPhase::Unknown);

        // Container statuses tell us how many containers are ready out of the total.
        // A pod might have multiple containers (sidecars, init containers, etc.).
        let container_statuses = pod.status.as_ref().and_then(|s| s.container_statuses.as_ref());

        let (ready_count, total_count) = container_statuses
            .map(|cs| {
                let total = cs.len();
                let ready = cs.iter().filter(|c| c.ready).count();
                (ready, total)
            })
            .unwrap_or((0, 0));
        let ready = format!("{ready_count}/{total_count}");

        // Sum restart counts across all containers in the pod.
        let restarts = container_statuses.map(|cs| cs.iter().map(|c| c.restart_count).sum()).unwrap_or(0);

        // Compute age from the creation timestamp.
        // k8s-openapi 0.27 wraps timestamps in a Time struct whose inner field (.0)
        // is a jiff::Timestamp. We subtract creation time from now to get the age.
        //
        // Important: older k8s-openapi versions used chrono::DateTime here.
        // If you see examples using chrono for K8s timestamps, they're outdated.
        let age = metadata
            .creation_timestamp
            .as_ref()
            .and_then(|ts| {
                let created = ts.0;                       // jiff::Timestamp
                let now = Timestamp::now();
                let diff = now.since(created).ok()?;      // jiff::SignedDuration
                Some(Duration::from_secs(diff.get_seconds().unsigned_abs()))
            })
            .unwrap_or_default();

        // The node the pod is scheduled on (may be None if still pending).
        let node = pod.spec.as_ref().and_then(|s| s.node_name.clone());

        Self { name, namespace, status, ready, restarts, age, node }
    }
}

// Human-readable duration formatting: "30s", "5m", "2h", "3d".
// Matches the style kubectl uses in its output.
pub fn format_duration(d: Duration) -> String {
    let secs = d.as_secs();
    if secs < 60 {
        format!("{secs}s")
    } else if secs < 3600 {
        format!("{}m", secs / 60)
    } else if secs < 86400 {
        format!("{}h", secs / 3600)
    } else {
        format!("{}d", secs / 86400)
    }
}
```

### Tests

```rust
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn pod_phase_display() {
        assert_eq!(PodPhase::Running.to_string(), "Running");
        assert_eq!(PodPhase::Failed.to_string(), "Failed");
        assert_eq!(PodPhase::Unknown.to_string(), "Unknown");
    }

    #[test]
    fn pod_summary_columns_returns_seven_entries() {
        let summary = PodSummary {
            name: "nginx".into(),
            namespace: "default".into(),
            status: PodPhase::Running,
            ready: "1/1".into(),
            restarts: 0,
            age: Duration::from_secs(3600),
            node: Some("node-1".into()),
        };
        let cols = summary.columns();
        assert_eq!(cols.len(), 7);
        assert_eq!(cols[0], ("NAME", "nginx".into()));
        assert_eq!(cols[2], ("STATUS", "Running".into()));
        assert_eq!(cols[5], ("AGE", "1h".into()));
    }

    // Verify the trait is object-safe — i.e., we can use Box<dyn ResourceSummary>.
    // This is important because future resource types (Deployment, Service, etc.)
    // will all implement this trait and may be stored in heterogeneous collections.
    #[test]
    fn resource_summary_trait_is_object_safe() {
        let summary = PodSummary {
            name: "test".into(),
            namespace: "default".into(),
            status: PodPhase::Pending,
            ready: "0/1".into(),
            restarts: 2,
            age: Duration::from_secs(120),
            node: None,
        };
        let boxed: Box<dyn ResourceSummary> = Box::new(summary);
        assert_eq!(boxed.name(), "test");
        assert_eq!(boxed.status_display(), "Pending");
    }

    #[test]
    fn format_duration_ranges() {
        assert_eq!(format_duration(Duration::from_secs(30)), "30s");
        assert_eq!(format_duration(Duration::from_secs(90)), "1m");
        assert_eq!(format_duration(Duration::from_secs(7200)), "2h");
        assert_eq!(format_duration(Duration::from_secs(172800)), "2d");
    }
}
```
